# M20机器狗上位机控制系统

## 项目结构

```
.
├── index.html              # 前端Web页面
├── backend/
│   ├── package.json        # Node.js依赖配置
│   ├── server.js           # 主服务器文件
│   ├── m20Client.js        # M20机器狗TCP客户端
│   └── m20Protocol.js      # M20通信协议处理
└── README.md               # 本文件
```

## 功能特性

### 前端功能
- **状态查询** - 实时显示机器狗状态（连接、运动、电池、位置等）
- **运动控制** - 站立/行走/趴下、步态切换、虚拟摇杆控制
- **导航控制** - 下发导航任务、定位初始化、自主充电
- **灯光控制** - 前后灯开关
- **通信日志** - 实时显示所有命令和状态

### 后端功能
- **WebSocket服务器** - 与前端实时通信
- **M20 TCP客户端** - 与机器狗通信
- **协议编解码** - 处理M20通信协议（16字节头部 + JSON ASDU）
- **心跳机制** - 5秒间隔心跳保活
- **自动重连** - 连接断开时自动重连（指数退避策略）

## 快速开始

### 1. 安装依赖

```bash
cd backend
npm install
```

### 2. 启动后端服务

```bash
npm start
```

服务器将在 `http://localhost:8080` 启动WebSocket服务

### 3. 打开前端页面

在浏览器中打开 `index.html` 文件

### 4. 连接到后端

- 在页面顶部输入后端服务器地址（默认 `127.0.0.1:8080`）
- 点击"连接"按钮

### 5. 连接到M20机器狗

后端会自动尝试连接到M20机器狗（`10.21.31.103:30001`）

## 配置说明

### 后端配置

编辑 `backend/server.js` 中的配置：

```javascript
const server = new ControllerServer({
    wsPort: 8080,              // WebSocket服务端口
    m20Host: '10.21.31.103',   // M20机器狗IP
    m20Port: 30001             // M20 TCP端口
});
```

### 前端配置

在页面顶部输入框中配置：
- **服务器IP** - 后端服务器地址
- **服务器端口** - 后端服务器端口

## 通信协议

### 前端 ↔ 后端 (WebSocket JSON)

#### 运动控制
```json
{
    "type": "motion_control",
    "action": "stand|walk|down"
}
```

#### 步态切换
```json
{
    "type": "gait_switch",
    "gait": "0x1001|0x1002|0x1003|0x3002|0x3003"
}
```

#### 灯光控制
```json
{
    "type": "light_control",
    "light": "front|rear",
    "state": true|false
}
```

#### 导航任务
```json
{
    "type": "nav_task",
    "payload": {
        "Value": 0,
        "PointInfo": 1,
        "PosX": 0.0,
        "PosY": 0.0,
        "PosZ": 0.0,
        "AngleYaw": 0.0,
        "Gait": "0x3002",
        "Speed": 0,
        "Manner": 0,
        "NavMode": 0,
        "ObsMode": 0
    }
}
```

#### 心跳
```json
{
    "type": "heartbeat",
    "timestamp": 1234567890
}
```

### 后端 ↔ M20 (TCP 二进制)

#### APDU结构
- 字节0-3: 协议头 `0xEB91EB90`
- 字节4-5: ASDU长度（小端字节序）
- 字节6-7: 消息ID
- 字节8: 数据格式（0x01=JSON）
- 字节9-15: 预留
- 字节16+: ASDU数据（JSON格式）

#### 心跳指令
```json
{
    "CommandCode": "0x0001",
    "Items": {}
}
```

## 重连策略

### 前端重连
- 连接断开时自动重连
- 重连延迟：1s → 2s → 4s → 8s → ... → 30s（指数退避）
- 最多重连10次

### 后端重连
- M20连接断开时自动重连
- 重连延迟：1s → 2s → 4s → 8s → ... → 30s（指数退避）
- 最多重连10次

## 故障排查

### 前端无法连接到后端
1. 检查后端服务是否启动
2. 检查防火墙设置
3. 检查IP和端口是否正确

### 后端无法连接到M20
1. 检查M20机器狗是否开启
2. 检查网络连接
3. 检查IP地址是否正确（默认 `10.21.31.103`）
4. 查看后端日志了解详细错误信息

### 命令无响应
1. 检查连接状态
2. 查看通信日志
3. 检查M20机器狗状态

## 日志输出

### 前端日志
- 在页面底部"通信日志"面板中显示

### 后端日志
- 在控制台输出
- 包括连接状态、命令处理、错误信息
