# M20机器狗上位机控制系统

云深处 M20 机器狗的单页面 Web 控制系统，提供实时状态查询、运动控制与导航等功能。

## 🎯 核心功能

- **状态监测**: 实时显示机器狗电量、位置(GPS)、CPU状态、运动状态等。
- **运动控制**: 支持使用模式切换(常规/导航/辅助)、步态切换、姿态控制及虚拟摇杆。
- **导航任务**: 支持单点导航下发、定位初始化及自主充电控制。
- **设备控制**: 支持前后照明灯控制。

## 🚀 快速启动

**推荐方式：使用一键脚本启动**

双击运行项目根目录下的系统启动脚本，会自动拉起 Node 后端并在浏览器中打开控制台：
- **Mac**: 双击运行 `start.command`
- **Windows**: 双击运行 `start_windows.bat`
- **Linux**: 执行 `./start_linux.sh`

*(按 `Ctrl+C` 或关闭终端即可停止服务)*

---

### 手动启动 (可选)
如果一键脚本无法使用，可以手动启动：

1. 安装依赖并运行后端：
```bash
cd backend
npm install
npm start
```
2. 在浏览器中打开项目根目录的 `index.html`。

### 连接机器狗

1. 前端页面打开后会自动连接后端（`localhost:8080`）。
2. 在顶部面板确认机器狗 IP 地址正确（默认 `10.21.31.103:30001`）。
3. 点击“连接”按钮建立 TCP 通信。

## ⚙️ 配置说明

### 后端配置

编辑 `backend/server.js` 中的配置参数：

```javascript
const server = new ControllerServer({
    wsPort: 8080,              // WebSocket服务端口
    m20Host: '10.21.31.103',   // M20机器狗IP地址
    m20Port: 30001             // M20 TCP端口
});
```

### 前端配置

在页面顶部配置M20地址：
- **M20地址** - 输入机器狗IP（默认 10.21.31.103）
- **M20端口** - 输入TCP端口（默认 30001）
- **连接** - 点击按钮连接M20

## 🔌 通信协议

### 前端 ↔ 后端 (WebSocket JSON)

#### 连接建立
```json
{
  "type": "connected",
  "message": "已连接到控制服务器"
}
```

#### 配置M20
```json
{
  "type": "config_m20",
  "host": "10.21.31.103",
  "port": 30001
}
```

#### 运动控制
```json
{
  "type": "motion_control",
  "action": "stand|walk|down"
}
```

#### 步态切换
```json
{
  "type": "gait_switch",
  "gait": "0x1001|0x1002|0x1003|0x3002|0x3003"
}
```

#### 灯光控制
```json
{
  "type": "light_control",
  "light": "front|rear",
  "state": true|false
}
```

#### 导航任务
```json
{
  "type": "nav_task",
  "payload": {
    "Value": 0,
    "PointInfo": 1,
    "PosX": 5.0,
    "PosY": 3.0,
    "PosZ": 0.0,
    "AngleYaw": 0.0,
    "Gait": "0x3002",
    "Speed": 0,
    "Manner": 0,
    "NavMode": 0,
    "ObsMode": 0
  }
}
```

#### 心跳
```json
{
  "type": "heartbeat",
  "timestamp": 1234567890
}
```

### 后端 ↔ M20 (TCP 二进制)

详见 [M20通信协议文档](docs/m20protocol.md)

**APDU结构**:
- 字节0-3: 协议头 `0xEB91EB90`
- 字节4-5: ASDU长度（小端字节序）
- 字节6-7: 消息ID
- 字节8: 数据格式（0x01=JSON）
- 字节9-15: 预留
- 字节16+: ASDU数据（JSON格式）

## 🔄 连接管理

### 自动重连策略

**前端 → 后端**:
- 连接断开时自动重连
- 重连延迟：1s → 2s → 4s → 8s → ... → 30s（指数退避）
- 最多重连10次

**后端 → M20**:
- 连接断开时自动重连
- 重连延迟：固定2秒
- 最多重连10次

### 心跳保活

| 方向 | 间隔 | 用途 |
|------|------|------|
| 前端 → 后端 | 5秒 | 保持WebSocket连接 |
| 后端 → M20 | 5秒 | 保持TCP连接 |

## 🛠️ 故障排查

### 前端无法加载

**问题**: 打开index.html显示空白或错误

**解决方案**:
1. 检查浏览器控制台（F12）是否有错误
2. 确保使用现代浏览器
3. 尝试使用本地服务器而不是直接打开文件

```bash
# 使用Python启动本地服务器
python -m http.server 8000
# 访问 http://localhost:8000
```

### 后端无法启动

**问题**: `npm start` 报错

**解决方案**:
1. 检查Node.js版本：`node --version`（需要v14+）
2. 重新安装依赖：`rm -rf node_modules && npm install`
3. 检查端口8080是否被占用：`lsof -i :8080`

### 前端无法连接后端

**问题**: 页面显示"✗ 未连接"

**解决方案**:
1. 确保后端服务已启动：`npm start`
2. 检查后端是否监听8080端口
3. 检查防火墙设置
4. 查看浏览器控制台错误信息

### 后端无法连接M20

**问题**: 日志显示"连接M20失败"

**解决方案**:
1. 检查M20机器狗是否开启
2. 检查网络连接：`ping 10.21.31.103`
3. 检查IP地址是否正确
4. 检查TCP端口30001是否开放
5. 查看后端日志了解详细错误

```bash
# 查看后端日志
npm start 2>&1 | tee server.log
```

### 命令无响应

**问题**: 发送命令后没有反应

**解决方案**:
1. 检查连接状态是否为"✓ 已连接"
2. 查看通信日志是否有错误信息
3. 检查M20机器狗状态
4. 尝试重新连接M20

### 虚拟摇杆不工作

**问题**: 拖拽摇杆没有反应

**解决方案**:
1. 确保机器狗处于"行走"状态
2. 检查浏览器是否支持触摸事件
3. 查看通信日志是否有发送命令
4. 尝试刷新页面

## 💡 使用技巧

### 1. 快速导航

使用键盘快捷键提高效率：
- 修改M20地址后，直接点击"连接"按钮
- 使用虚拟摇杆进行精细控制

### 2. 导航任务配置

**直线导航** - 适合开阔场景：
- 导航方式：直线导航
- 步态：平地（敏捷）
- 速度：正常或低速

**自主导航** - 适合复杂地形：
- 导航方式：自主导航
- 步态：平地（敏捷）
- 速度：低速（必须）

**楼梯导航** - 特殊配置：
- 导航方式：直线导航
- 步态：楼梯（敏捷）
- 停避障：关闭
- 目标点：距离楼梯边缘45cm以外

### 3. 性能优化

- 虚拟摇杆以20Hz频率发送（50ms）
- 心跳以5秒频率发送
- 避免频繁切换步态
- 导航任务完成后再下发新任务

### 4. 调试技巧

打开浏览器开发者工具（F12）：
- **Console** - 查看JavaScript错误
- **Network** - 查看WebSocket通信
- **Application** - 查看本地存储
